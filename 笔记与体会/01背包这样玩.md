# 01 背包
## 认识 01 背包
01背包核心规则：每件物品**最多只能选一次**，对应两种操作：选 / 不选。
常规场景是给定重量、价值，求容量限制下最大价值；但可以变形拓展，结合**二进制状态压缩**解决集合覆盖类最小选取问题，典型例题：[H鸡蛋短缺](../03-NowCoder/比赛/酱菜暑期集训/H鸡蛋短缺.md)。

## 基础玩法
### 1. 标准一维01背包核心逻辑
一维滚动数组优化后，内层循环必须**逆序遍历**容量/状态，本质目的：保证更新状态时，使用的是上一轮未被污染的旧状态，防止同一物品重复选取。

### 2. 针对 H.鸡蛋短缺 的特殊变形
常规背包以「重量」为维度，本题以**二进制位掩码 state** 作为dp数组下标：
- `dp[state]`：凑出鸡蛋集合状态 `state` 需要的最少套装数量；
- 每件物品（套装）作用：按位或扩充现有集合，而非累加重量；
- 转移式：`new_state = state | a[i]`，取更小套数更新 `dp[new_state] = min(dp[new_state], dp[state] + 1)`。

### 注意事项
绝大多数时候，01背包dp的循环更新是从后往前遍历，原因分两类：
1. 传统重量背包：dp[j] 更新依赖本轮左侧更小容量旧值，正序会重复选取物品；
2. 位掩码状态背包（H鸡蛋短缺属于此类）：循环到物品 i 时，更新会生成更大的 state，若正序遍历会使用本轮刚更新完成的 state，造成套装重复选择，破坏01背包“每套只能选一次”的约束。

例题对照：[H鸡蛋短缺](../03-NowCoder/比赛/酱菜暑期集训/H鸡蛋短缺.md)
```cpp
// H鸡蛋短缺核心背包代码
vector<int> dp(up+1, m+1);
dp[0] = 0;
// 遍历每件套装（物品）
for(int i=0; i<m; i++) {
    // 逆序遍历所有状态，防止重复选取同一套装
    for(int state=up; state>=0; state--) {
        if(dp[state] != m+1) {
            int new_state = state | a[i];
            dp[new_state] = min(dp[new_state], dp[state] + 1);
        }
    }
}
```

## 拓展：两种背包遍历对比
| 背包类型 | 内层遍历顺序 | 适用场景 | 对应本题逻辑 |
| ---- | ---- | ---- | ---- |
| 01背包（本题） | 逆序从大到小 | 物品至多选1次 | 每套鸡蛋套装只能买一次 |
| 完全背包 | 正序从小到大 | 物品可无限选 | 本题不适用 |
