# G. 精度误差？！
> 做题时间：2026-07-17
> 竞赛：2026牛客暑期多校训练营1
> 原题链接：https://ac.nowcoder.com/acm/contest/133876/G

## 题目简述
给定整数 n，在三维空间中构造一个不超过 2n+2 个不同点的集合 S，要求对每个点 P∈S，恰好存在 n 个不同的点 Q∈S 满足两点欧氏距离为 1。
题目允许 ε=0.01 的精度误差：任意两点距离必须大于 ε；距离落在 (1-ε, 1+ε) 区间内即视为距离等于 1。输出任意一组合法点集即可。

## 利用精度误差的构造思路
如果严格构造正多面体求解，对几何推导能力要求较高。但本题有一个可利用的关键条件：两点距离只要落在误差区间内就会被判为合法。结合点集上限为 2n+2，我们可以采用双平面的简化构造方案：将 n 个点放在 z=0 平面，另外 n 个点放在 z=1 平面，只要保证每个点与对面平面所有点的距离都落在误差范围内，就能满足「每个点恰好有 n 个距离约为 1 的点」的要求。

我们以点 (0,0,0) 为例计算对面平面的合法取点范围：已知 ε=0.01，两点最大允许距离为 1.01，因此 z=1 平面上的点到 z 轴的水平距离最大值为 √(1.01² - 1²) = √0.0201 ≈ 0.141774。进一步，为了让两平面内任意两点的跨平面距离都满足误差要求，我们可以在两个平面上各取一个半径约 0.07088 的同轴小圆，只要点都落在各自小圆内，任意跨平面对应点的距离就都会在合法区间内。

接下来处理同平面内的点间距约束：要求任意两点距离大于 ε=0.01。我们采用正方形网格密铺的方式，取相邻点间距 d=0.01001，在圆的直径行可以放下 14 个点，向上每层依次减少 2 个（12、10、8…），向下与上半部分对称排布，总点数完全可以覆盖 n≤100 的数据范围。

输出时按层遍历网格点，依次输出坐标，直到同一平面内输出满 n 个点即可。我采用的遍历顺序是：先输出直径行，再交替输出上方行与下方行，逐层向内收缩。

## 完整代码
```cpp
#include <bits/stdc++.h>
#define QwQ cin.tie(0)->sync_with_stdio(0);
using namespace std;
const double d = 0.01001;

void go() {
    int n;
    cin >> n;
    cout << 2 * n << '\n';
    // 按正方形密布方式输出指定 z 平面的 n 个点
    auto printp = [&](double z) -> void {
        double st = 0;
        double y = 0;
        int res = n, k = 14;
        // res 剩余待输出的点数，k 为当前行可放置的点数（直径行最多14个）
        int i = 0; // 当前行已输出的点数计数器
        while (i < res) {
            // 当前行已输出满k个点，切换到下一行
            if (i != 0 && i % k == 0) {
                res -= k; // 扣除已输出的点数
                i = 0;  // 重置行内计数器，准备输出新一行
                if (y <= 0) {
                    y = y * (-1) + d;
                    if (k > 2)
                        k -= 2;
                    else
                        k--;
                    st += d;
                } else {
                    y *= -1;
                }
            }
            cout << st + i * d << ' ' << y << ' ' << z << '\n';
            i++;
        }
    };
    cout << fixed << setprecision(10);
    printp(0.0);
    printp(1.0);
}

int main() {
    QwQ int t = 1;
    cin >> t;
    while (t--) {
        go();
    }
}
```

## 心得
这是一道典型的 Checker Hack 构造题。以后遇到带精度容差的构造类题目，不必强求严格的几何正解，可以多思考如何利用误差范围简化构造思路，用更轻松的方式通过校验。
