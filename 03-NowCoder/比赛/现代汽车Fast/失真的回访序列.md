# 失真的回访序列
> 做题时间：2026-07-16
> 竞赛：2026现代汽车急速编程挑战
> 原题链接：https://acexam.nowcoder.com/coding/?uid=EA95895BF192DAE1&qid=11694678

## 题目简述
计数器初始值为 0，共有 n 次操作，第 i 次操作会给计数器加上整数 $a_i$。

回放机存在故障：它会**任选一段长度恰好为 k 的连续操作**，将这段操作重复执行一遍——也就是该段操作对计数器的贡献会额外增加一次。

我们可以进行最多 m 次修改：每次选择一个操作，将对应的 $a_i$ 加 1。

我们的目标是：无论机器选择哪一段长度为 k 的连续操作重复执行，最终计数器的结果都尽可能大。请求出**最终计数器最小可能值的最大值**（即最大化最坏场景下的计数器数值）。

## 我的想法
审完题目思考一会儿之后，我第一时间想到的是定长滑动窗口，窗口长度固定为 k，思路是找到求和最小的窗口，给它加上增量 m。
但转念一想，给窗口加完数值后，又会诞生其他求和值更小的窗口，无法一次性平衡所有区间。
我想把数组所有窗口按和从小到大依次补增量，让各个窗口总和趋于平均，但实操时代码逻辑非常复杂，存在几个难以解决的卡点：
1. 如何批量存储全部窗口信息；
2. 每次增量应该加到窗口内哪一个元素；
这套思路落地难度极高，写到一半直接卡住，找不到可行实现方案。

## 破局之法
参考了二分答案 + 滑动窗口的解法后，这个问题有了另一种思考方式。

既然逐个窗口加增量难以模拟实现，而最终所有窗口的和都会趋近于一个目标值，不如直接二分猜测这个目标值，然后判断在总修改次数不超过 m 的前提下，能否让所有长度为 k 的连续子数组和都不小于这个目标值：
- 若让所有窗口达到目标值所需的总增量 > m，说明答案猜大了，缩小右边界；
- 若总增量 ≤ m，说明答案可行，尝试更大的目标，扩大左边界。

由此形成了天然的二分框架。

> 注意：当窗口求和小于猜测的答案时，为窗口补充增量时，贪心地加在窗口最右端的元素上，让后续重叠的窗口都能复用这份增量，最小化总修改次数。

## 完整代码
```cpp
#include<bits/stdc++.h>
#define int long long
using namespace std;

void go() {
    int n, k, m;
    cin >> n >> k >> m;
    vector<int> a(n+2);
    vector<int> pre(n+2,0);
    int sum = 0;
    for(int i=1; i<=n; i++) {
        cin >> a[i];
        sum += a[i];
        pre[i] = pre[i-1] + a[i];
    }
    
    int l = -2e14, r = 2e14;
    int mn;
    // 校验函数：判断目标mid是否可以用不超过m的增量实现
    auto check = [&](int mid) -> bool {
        vector<int> ad(n+2, 0);
        int cur = 0;    // 当前窗口内叠加的增量总和
        int total = 0;  // 累计消耗的总增量
        for(int i=1; i<=n-k+1; i++) {
            int ri = i + k - 1;
            // 窗口左边界移出，减去对应增量
            if(i > 1) cur -= ad[i-1];
            int s = pre[ri] - pre[i-1];
            int now = s + cur; // 当前窗口真实总和
            if(now < mid) {
                int de = mid - now;
                ad[ri] += de; // 增量加在窗口最右端，复用至后续重叠窗口
                cur += de;
                total += de;
                if(total > m) return false;
            }
        }
        return total <= m;
    };
    // 二分最大化最小窗口和
    while(l <= r) {
        int mid = l + (r - l) / 2;
        if(check(mid)) {
            mn = mid;
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }
    cout << sum + m + mn << '\n';
}

signed main() {
    cin.tie(0)->sync_with_stdio(0);
    int t; cin >> t;
    while(t--) go();
    return 0;
}
```

## 心得
当答案需要通过数值增减来达成、且直接模拟加减操作的实现难度很高时，可以换个思路：尝试二分猜测最终答案，再通过校验函数判断答案是否合法，以此不断收缩范围找到最优解。

二分答案非常适合“操作模拟复杂，但校验条件容易实现”的题型。
