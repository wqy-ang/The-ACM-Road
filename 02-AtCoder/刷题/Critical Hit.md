# 模意义下期望DP与分数运算详解
> 适用于算法竞赛中「答案对质数取模」的概率/期望DP类题目，以经典攻击怪兽期望次数问题为例。

---

## 一、问题背景
在算法竞赛中，很多概率、期望类题目的真实答案是有理数（分数），直接使用浮点数计算会存在精度误差，因此约定：
- 将最终答案表示为最简分数 $\frac{P}{Q}$
- 在模质数意义下，用一个整数 $R$ 等价表示该分数
- 满足 $R \times Q \equiv P \pmod{MOD}$，输出 $R$ 即可

所有加法、乘法、除法运算都可以在模意义下转化为整数运算，全程无精度损失。

---

## 二、核心概念：模逆元
### 1. 为什么需要逆元
模运算世界中**不存在除法**，不能直接计算 $\frac{a}{b} \bmod MOD$。
我们引入「模逆元」作为等价替代：
> 除以一个数，等价于乘以这个数的模逆元

### 2. 定义
若整数 $b$ 与模数 $MOD$ 互质，则存在唯一整数 $inv(b)$，满足：

$b \times inv(b) \equiv 1 \pmod{MOD}$

此时有：

$\frac{a}{b} \equiv a \times inv(b) \pmod{MOD}$

### 3. 费马小定理求逆元
当 $MOD$ 为质数时，由费马小定理可得：

$inv(x) = x^{MOD-2} \pmod{MOD}$

通过快速幂（`qpow`）即可在 $O(\log MOD)$ 时间内计算逆元。

---

## 三、关键代码行解析
### 代码片段
```cpp
p = 1ll * p * qpow(100) % mod;
```

### 含义解释
这行代码的作用是**将概率 $\frac{P}{100}$ 转换为模意义下的整数形式**。
- 原始概率为 $\frac{P}{100}$
- 等价于 $P \times inv(100) \pmod{998244353}$
- `qpow(100)` 计算的就是 100 在模 998244353 下的逆元

### 补充细节
- `1ll` 是长整型字面量，用于强制类型提升，避免 32 位整数乘法溢出
- 末尾的 `% mod` 保证结果始终落在模数范围内

### 延伸：1-p 的模运算写法
```cpp
(mod + 1 - p) % mod
```
- 用于计算 $1-p$ 的模值
- 先加 `mod` 再取模，是为了避免减法导致负数结果

---

## 四、题目中 R 的实际意义
### 定义
题目要求输出的 $R$，就是分数答案在模 998244353 下的整数代表值。

若真实答案为最简分数 $\frac{P}{Q}$，则 $R$ 满足：

$R \times Q \equiv P \pmod{998244353},\quad 0 \le R < 998244353$

### 举例
- 模数取 7，答案为 $\frac{1}{2}$
- 因为 $4 \times 2 = 8 \equiv 1 \pmod 7$，所以 $inv(2)=4$
- 因此 $R = 1 \times 4 \equiv 4 \pmod 7$，输出整数 4 即可代表分数 $\frac{1}{2}$

## 五、期望DP求解 R 的完整思路
以「怪兽体力为 $N$，每次攻击 $p$ 概率扣2血，$1-p$ 概率扣1血，求期望攻击次数」为例。

### 1. 状态定义
`dp[i]`：当前怪兽剩余体力为 $i$ 时，到体力降为 0 所需的期望攻击次数（模意义下的 R 值）。

### 2. 边界条件
当体力 $\le 0$ 时，攻击停止，期望次数为 0：

$dp[i] = 0 \quad (i \le 0)$

### 3. 状态转移方程
根据期望的线性性，攻击 1 次后有两种情况：
- 概率 $1-p$：扣 1 血，转移到 $i-1$
- 概率 $p$：扣 2 血，转移到 $i-2$

因此转移方程为：

$dp[i] = 1 + (1-p) \cdot dp[i-1] + p \cdot dp[i-2] \pmod{MOD}$

式中的 $+1$ 代表本次攻击消耗的 1 次操作。

### 4. 计算顺序
从 $i=1$ 正向递推到 $i=N$，最终 `dp[N]` 即为题目要求的 R 值。
> 也可以反向定义状态（从 0 血到 i 血），从后往前递推，逻辑完全等价。

## 六、完整可AC代码
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;

const long long mod = 998244353;

// 快速幂
int qpow(int x, int y) {
    int a = 1;
    while (y) {
        if (y & 1)
            a = (a * x) % mod;
        x = (x * x) % mod;
        y >>= 1;
    }
    return a;
}

signed main() {
    int n, p, q;
    cin >> n >> p;

    // 将概率 p/100 转换为模意义下的整数
    p = p*qpow(100, mod - 2) % mod;
    q = (mod + 1 - p) % mod;

    vector<int> dp(n + 2, 0);
    for (int i = n - 1; i >= 0; i--) {
        // 状态转移方程
        dp[i] = (1 + q * dp[i + 1] % mod + p * dp[i + 2] % mod) % mod;
    }

    cout << dp[0] << '\n';

    return 0;
}
```

---

## 七、注意事项
1. **数据类型**：乘法中间结果可能超出 int 范围，必须使用 `long long` 存储，乘完后再取模。
2. **负数处理**：所有减法运算后都要加 MOD 再取模，保证结果非负。
3. **数组越界**：DP 数组大小建议开成 `MAXN + 10`，避免边界下标越界。
4. **逆元前提**：费马小定理仅适用于模数为质数的情况，998244353、1e9+7 均为质数，可直接使用。
5. **0 值逆元**：0 不存在逆元，分母不能为 0。
