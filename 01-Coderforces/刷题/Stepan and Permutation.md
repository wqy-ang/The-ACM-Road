# C. Stepan and Permutation

> 原题链接：https://codeforces.com/contest/2244/problem/C

## 题目简述
给定一个长度为 $n$ 的排列 $p$，以及两个正整数 $x$ 和 $y$。规定每次操作可以选择两个下标，若它们的差的绝对值等于 $x$ 或 $y$，即可交换对应位置的元素。请问能否通过任意次这样的操作，将排列按升序排列。

## 思路简述
本题核心是**并查集连通性判定**：
1.  可交换的位置具有传递性：如果位置 $a$ 能和 $b$ 交换，$b$ 能和 $c$ 交换，那么 $a,b,c$ 三个位置的元素可以任意排列。
2.  可以想到使用并查集把所有可以任意互换的位置合并在一个集合，在循环中用并查集维护 位置 $i$、 $i+x$ 和 $i+y$。
3.  由于输入是排列（元素互不重复），排序后第 $i$ 个位置的元素对应原数组唯一的下标。用 `vector<PII> pa(n)` 记录初始数值和它的位置，只要每个目标位置与对应元素的原下标在同一个连通块中，就说明可以通过交换完成排序。

## 错点
`merge` 函数中越界判断时机错误，先执行 `find` 访问数组，后判断下标是否合法，会导致数组越界访问，触发未定义行为。

```cpp
merge = [&](int x, int y) -> void {
    int rx = find(x);   // 先执行find
    int ry = find(y);
    if(rx == ry || x >= n || y >= n) return; // 后判断越界
    ...
};
```

执行find时，x和y可能已经越界，所以要先判断

修正后：
```cpp
merge = [&](int x, int y) -> void {
    if(x >= n || y >= n) return; // 先判越界
    int rx = find(x);
    int ry = find(y);
    if(rx == ry) return;
    if(abs(fa[rx]) < abs(fa[ry])) swap(rx, ry);
    fa[rx] += fa[ry];
    fa[ry] = rx;
};
```

## 反思
所有存在下标越界可能的场景，务必先做合法性判断，再执行数组访问等后续逻辑，从根源上杜绝越界访问隐患。
